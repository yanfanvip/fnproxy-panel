# 证书管理与自动续签

## 背景

为管理后台补充 HTTPS 证书生命周期管理能力，支持：

- 证书列表查看
- ACME 自动申请与自动续签
- 上传文件导入已有证书
- 通过外部 JSON 配置文件同步导入证书
- 支持文件校验（HTTP-01）和 DNS 校验（DNS-01）
- DNS 校验至少支持腾讯云、阿里云、Cloudflare

## 方案概览

- 后端 ACME 能力基于 `github.com/go-acme/lego/v4` 实现。
- 证书配置存储在 `fnproxy-panel.json` 的 `certs` 节点。
- 外部证书配置文件路径与同步周期存储在全局配置中。
- 运行时支持通过启动参数 `-config_path` 指定配置/缓存/证书/PID 文件的保存根目录，默认使用当前运行目录。
- 程序启动时会先检查 `<config_path>/fnproxy-panel.pid` 对应进程是否仍在运行；若已运行则直接退出，避免重复启动。
- 证书文件落盘到：
  - `certs/managed/<id>.crt`
  - `certs/managed/<id>.key`
  - `certs/accounts/<id>.account.key`
- 运行中的进程 PID 会写入 `<config_path>/fnproxy-panel.pid`。
- HTTPS 监听从“按端口固定证书”切换为“按 SNI 动态选证书”。
- 程序内嵌一份 100 年有效期的默认证书与私钥；当服务未匹配到可用业务证书时，会优先使用该内嵌默认证书作为 HTTPS 回退证书。
- 自动续签任务在进程内定时执行，默认每小时扫描一次。

## 页面与接口

新增“证书管理”页面，表格展示：

- 域名
- 来源
- 校验方式
- DNS 服务商
- 有效期
- 续签日期
- 状态

页面支持：

- 新建证书
- 编辑证书
- 通过文件上传导入证书
- 通过设置页配置外部证书 JSON 文件路径
- 手动立即续签
- 删除证书
- 在 HTTPS 服务配置中显式选择证书

新增接口：

- `GET /api/certificates`
- `GET /api/certificates/{id}`
- `POST /api/certificates`
- `PUT /api/certificates/{id}`
- `POST /api/certificates/{id}/renew`
- `DELETE /api/certificates/{id}`

## 校验方式说明

### 文件校验（HTTP-01）

- 依赖当前面板已启用 `HTTP 80` 端口监听。
- 域名需要解析到当前服务器。
- 运行时通过 `/.well-known/acme-challenge/` 动态响应 ACME 校验请求。

### DNS 校验（DNS-01）

已支持：

- 腾讯云 DNSPod
- 阿里云 AliDNS
- Cloudflare

## 外部配置文件同步

设置页新增：

- `外部证书配置文件路径`
- `外部证书同步周期（秒）`

默认值：

- 外部证书配置文件路径：`/usr/trim/etc/network_gateway_cert.conf`
- 外部证书同步周期：`3600` 秒（1 小时）

配置文件格式：

```json
[
  {
    "host": "fnOS",
    "cert": "/path/to/fnOS.crt",
    "key": "/path/to/fnOS.key"
  },
  {
    "host": "geekcode.com.cn",
    "cert": "/path/to/fullchain.crt",
    "key": "/path/to/domain.key"
  }
]
```

行为说明：

- 系统会按配置周期重新读取该 JSON 文件。
- 若配置文件不存在，系统会静默跳过本轮同步，不报错、不影响程序运行。
- 若证书文件或私钥文件的修改时间发生变化，会自动重新加载证书。
- 同一个 `host` 在同一个配置文件中会被视为同一条证书配置：
  - 已存在则比较并更新证书配置
  - 不存在则新增证书配置
- 外部配置文件中的证书会以 `配置文件` 来源展示在证书管理页面。
- 若某条外部证书从 JSON 文件中移除，系统会自动移除对应证书并清空相关服务的显式绑定。
- 外部配置文件同步证书不能在页面里单独编辑或删除，应直接修改源配置文件。

## 运行时行为

- 启动时加载已存在证书到内存。
- TLS 握手阶段优先根据“监听器下服务规则的显式证书绑定 + SNI”选择证书。
- 若服务未显式绑定证书，则回退为根据证书自身域名列表匹配 SNI。
- 若没有匹配证书，则使用进程内生成的兜底自签名证书，避免 HTTPS 监听因缺失证书直接启动失败。

## 自动续签

- 仅 ACME 证书支持自动续签。
- 每张 ACME 证书可单独配置：
  - 是否启用自动续签
  - 提前多少天触发续签
- 后台进程启动后会在 10 秒后做一次初始扫描，之后按“外部证书同步周期”执行维护扫描。
- 到达 `next_renew_at` 后会自动执行续签，并刷新证书文件、内存缓存和到期时间。
- 外部配置文件证书也会按同步周期检查文件变更并刷新运行时缓存。

## 已知限制

- 当前“文件校验”未内置独立临时 `:80` 挑战服务器，因此必须由现有监听器承接 80 端口流量。
- 当前不支持直接把“导入证书”和“ACME 证书”互相切换类型；若需要切换，建议删除后重新创建。
