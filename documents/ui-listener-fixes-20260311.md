# UI And Listener Fixes

## 背景

本次修复聚焦 3 个已确认问题：

1. 端口监听列表点击编辑时，默认响应规则无法修改。
2. 页面整体视觉较粗糙，弹出层交互体验较弱。
3. 端口详情页面在切换查看不同端口时，存在显示串端口数据的情况。

## 本次改动

- 优化 `src/static/index.html` 页面样式，统一卡片、按钮、表格、输入框、模态框、确认框和提示消息的视觉风格。
- 在 `src/static/app.js` 中重构监听编辑弹窗的默认响应规则逻辑，支持：
  - 编辑已有默认规则
  - 替换默认规则类型
  - 删除默认规则
  - 在编辑场景下新增默认规则
- 重构服务弹窗的表单回填与模式切换逻辑，修复编辑服务时配置不回显的问题。
- 为端口详情服务列表增加请求令牌和端口二次过滤，避免快速切换端口时旧请求覆盖当前页面数据。
- 更新 `src/config/manager.go`，删除端口监听时同步清理该端口下的服务配置，避免遗留孤儿数据。
- 更新 `src/fnproxy/server.go` 的监听路由逻辑，改为基于请求 `Host` 匹配服务域名规则（含 `*` 通配），修复 `a.localhost -> 8443 -> 8100` 场景下默认文本输出返回 404 的问题。
- 新增运行时监控能力：
  - 首页使用 24 小时网络流量图表展示每 10 分钟平均值
  - 端口监听页展示连接数、请求数、总流量和实时速率
  - 服务配置页展示服务级流量与连接统计
- 新增访问日志能力：
  - 按服务配置项 `access_log` 控制是否记录日志
  - 监听页和服务页分别提供日志查看区域
- 新增服务侧 OAuth 登录流程：
  - 服务启用 `oauth` 后，未登录访问会重定向到服务地址下的 `/OAuth`
  - OAuth 登录页采用移动端友好的新版 UI，并支持浏览器端公钥加密提交用户名/密码
  - 登录成功后写入认证 Cookie，并跳回原始请求地址
  - 若用户已经处于登录态，再次访问服务地址下的 `/OAuth`，则不再展示本地登录页，也不做额外跳转，而是直接放行给当前命中的服务按原始路径继续处理
  - 用户管理新增独立 `token` 配置项，可在界面中一键生成 32 位随机 token；当业务请求携带 `Auth` 请求头且 token 与某个已启用用户配置一致时，也会直接视为已登录
  - 用户管理操作区新增复制 token 按钮，便于直接将已配置的用户 token 用于 `Auth` 或 `Authorization: Bearer` 请求头调用
  - 记住我模式下 JWT 有效期为 30 天，默认模式为 1 天
  - 启动参数 `-secure` 会参与生成稳定的服务端加解密密钥，并作为用户密码摘要盐值使用；未显式传入时默认值为 `123456`
  - 用户管理接口按明文接收密码，服务端会基于 `secure` 生成确定性摘要后写入配置文件
  - OAuth 登录提交仍为前端公钥加密，服务端解密后会按同一 `secure` 规则生成摘要并与配置文件中的密码进行比对
  - 服务端日志会区分记录 OAuth 登录失败原因：解密失败、用户不存在、用户被禁用、密码错误、令牌生成失败
  - 默认 `admin/admin` 可用
- 调整首页状态卡片：
  - 运行时间按秒 / 分钟 / 小时 / 天+小时的整数规则展示
  - 内存占用改为展示当前程序进程的内存占用
  - 手机端首页统计卡片改为一行两个，并缩小卡片尺寸
- 修复流量速率显示偶发 `undefined/s` 的问题：
  - 前端字节单位格式化增加上限与非数值兜底
  - 后端网络速率计算在网卡计数回退时改为返回 `0`，避免无符号下溢导致异常大数值
- 调整监控与日志存储方式：
  - 访问日志与 24h 网络流量采样改为落盘到缓存文件 `cache/monitor-cache.db`
  - 页面读取日志与流量历史时优先从缓存文件查询，避免长期运行时占用过多内存
  - 实时连接数与分钟级速率窗口仍保留在内存中，用于保持界面实时性
  - 设置页新增 `日志保留天数` 与 `最大日志条数` 配置项，用于控制缓存文件中的日志保留范围
- 调整用户管理页面：
  - 添加用户按钮改为 icon 按钮
  - 新增用户字段调整为用户名、密码、Email、是否开启
  - 用户列表增加启用状态列与启用/禁用操作
  - 用户列表操作按钮改为 icon 按钮
  - 用户列表增加编辑功能，支持修改密码、Email、启用状态
  - 取消对 `admin` 用户的特殊删除/禁用限制
- 调整端口服务列表排序能力：
  - 服务规则支持拖拽排序，并持久化到配置文件
  - 默认规则固定保持在列表最下方，且不参与拖拽
  - 新增服务、编辑服务、运行时重载时均按该顺序生效
- 调整反向代理能力：
  - 反向代理现已支持 WebSocket / WSS 升级请求
  - 上游地址除了 `http://`、`https://` 外，也兼容直接填写 `ws://`、`wss://`
- 新增端口详情页“重载服务”按钮：
  - 点击后对当前端口执行热重载
  - 重载前先校验服务配置，配置有误时继续保留上一次成功运行的配置
  - 前端直接提示具体错误信息，避免静默失败
- 新增端口监听保存校验：
  - 创建和编辑端口时会先检查配置中是否已有相同端口
  - 同时检测端口是否已被其他程序占用
  - 若端口被其他程序占用，则允许保存监听与服务配置，但自动置为未启动状态
  - 若端口重复，则仍然阻止保存，并返回明确错误提示
  - 管理后台使用中的端口禁止被监听器占用
- 调整端口卡片启停逻辑：
  - 点击开启时，只有在端口校验通过且实际启动成功后，才会保存为启用状态
  - 若端口被占用、命中管理端口或启动失败，则仍保持关闭状态
- 调整端口卡片运行/配置状态逻辑：
  - 启动停止按钮按程序真实运行状态判断，而不是按 `enabled` 配置值判断
  - 点击开启时，即使启动失败，也允许保存 `enabled=true`
  - 卡片可同时体现“配置已启用”但“当前未运行”的状态
- 调整静态文件服务目录浏览行为：
  - 开启目录列表后，前端不再展示“首页文件/默认首页文件”设置项
  - 目录请求会直接返回自定义文件浏览器页面，不再优先展示 `index.html`
- 调整程序启动时的监听加载策略：
  - 管理后台端口会先绑定成功，再启动各个业务监听端口
  - 启动阶段按端口逐个尝试加载已启用监听
  - 某个端口启动失败时仅记录错误，不阻塞管理后台与其他端口继续启动
  - 控制台会输出失败端口及原因，便于排查占用冲突
- 新增启动参数：
  - `-config_path` 用于指定配置文件、缓存、证书和 PID 文件的保存根目录
  - `-port=数字` 可覆盖管理后台 TCP 端口
  - `-port=sock` 会在 `config_path` 下创建 `fnproxy-panel.sock` 作为管理后台 Unix Socket
  - `status` / `-action=status` 会根据 PID 文件判断程序是否运行
  - `stop` / `-action=stop` 会根据 PID 文件停止当前进程
  - `restart` / `-action=restart` 会先停止 PID 文件对应进程，再启动新进程
  - 正常启动时若 PID 文件对应进程仍存在，程序会直接退出，避免重复启动
  - 静态资源已在构建时内嵌进可执行文件，部署时无需再额外复制前端文件
  - 程序已内嵌默认 HTTPS 回退证书，未匹配到业务证书时可直接提供基础 TLS 能力
  - 根目录提供 `build.windows.bat`、`build.linux.sh`、`build.linux.bat`、`debug.bat` 用于构建与本地调试
- 调整 SSH 连接管理：
  - SSH 连接密码提交改为前端公钥加密，服务端使用 `secure` 派生密钥进行可逆加密后再保存
  - SSH 卡片新增编辑入口，支持修改名称、连接方式、地址、用户名、密码和工作目录
  - 点击 SSH 卡片时会优先弹出终端窗口；即使远程 SSH 无法连接，也会在弹窗内展示错误信息

## 验证

- 执行 `go test ./...`，通过。
- 实测 `https://a.localhost:8443/` 会先跳转 `/OAuth` 登录页，使用 `admin/admin` 登录后可正常访问后端服务。
- 实测监听统计、服务统计、访问日志接口均可返回真实运行数据。

## 备注

- 当前服务流量、连接数等指标仍为占位展示，后续若需要准确统计，需要补充实时采集与端口级聚合逻辑。
